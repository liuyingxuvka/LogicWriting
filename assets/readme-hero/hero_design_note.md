# Logic Writing README hero design note

- **Project summary:** one Codex skill that routes investigation, academic writing, fiction, or travel guidance through a shared reader/artifact kernel.
- **Target users:** people using Codex for difficult reports, papers, stories, novels, and evidence-heavy travel guides.
- **Core problem:** domain depth, logical structure, and reader-ready prose are often handled as separate or competing workflows.
- **Core workflow:** four final-owner routes retain their native semantics, pass through one neutral writing kernel, and converge into a `Reader-ready artifact` on the right.
- **Hero tagline:** Four final owners. One reader-ready artifact.
- **Visual concept:** an airy, dimensional evidence loom with four colored lanes, seven bounded specialist markers, and one current delivered artifact.
- **Image keywords:** four routes, bounded specialists, reader state, model-artifact binding, clear prose, light technical product render.
- **Files:** `assets/readme-hero/hero.png` and this note. The reproducible vector source is kept only under the ignored `work/` directory.
- **README position:** immediately below the title, public-safe badges, and language link.

The hero is deterministically rasterized because the exact route names and
artifact label are part of the explanation. This avoids unreadable generated
text and keeps the public asset reproducible; it is not a reusable card-grid
template.
